## Findings considered and rejected

*From the quick audit (2026-07-12):*

- **`modelContext` nil-drop in `PhoneSessionManager.insertMatch`**: not reachable in practice — the container is wired synchronously in `BeachTennisApp.init` on the main actor before any delegate callback can hop to `@MainActor`.
- **iOS `Color(hex:)` silently falling back to `.black`**: inputs are app-controlled constants from `SettingsView`'s fixed palette; no user-supplied hex exists.
- **`pushColorsToWatch`/`pushSettingsToWatch` duplication as its own plan**: folded into plan 005 instead (the method is dead code with zero callers).
- **Security category**: audited, no findings — no network surface beyond WatchConnectivity, no secrets, no interpreters fed by user input. (Re-confirmed by the deep audit over the full repo.)

*From the deep audit (2026-07-12):*

- **Unreachable 40-40 branch in `awardBeachTennisGamePoint`** (`ScoreEngine.swift:33-36`): sets `isGoldenPoint` and swallows the point, but is dead in practice — reaching 40-40 always sets the flag at lines 47-49 first; only externally constructed states hit it (tests do). Harmless defensive code; not worth a plan. Plan 002/006's executor may simplify it *only* if characterization tests exist.
- **`MatchResultPayload.date` is match end time, not `matchStartDate`** (`WatchSessionManager.swift:32`): "Date" in history means "when it finished" — defensible semantics, zero user complaints possible either way. Not worth churn.
- **Identical ternary `matchType == .tennis ? "Sets" : "Sets"`** (`ScoreView.swift:77`): cosmetic noise; the label happens to be right for both modes (beach UI intentionally calls games "Sets" per `CLAUDE.md`). Fix opportunistically, no plan.
- **Duplicate `@AppStorage("sportSetting")` in `SettingsView` and `PhoneSessionManager`**: same UserDefaults key, so behavior is correct; plan 005 already drops the redundant re-assignment. No further action.
- **One-line `README.md`**: real but lowest-value docs gap for a single-maintainer repo whose `CLAUDE.md` already documents build/architecture. Deferred, not planned.
- **Missing `CONTEXT.md`/`docs/adr/`**: per `docs/agents/domain.md`, absence is expected and explicitly not to be flagged — they're created lazily by other skills.
- **Performance category**: audited, no findings — no loops over unbounded data, no I/O on hot paths; the in-memory filter in `MatchListView` is fine at personal-history scale.
- **Dependencies category**: audited, no findings — zero third-party dependencies; toolchain is Xcode + XcodeGen only.
